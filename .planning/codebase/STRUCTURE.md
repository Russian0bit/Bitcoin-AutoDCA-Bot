## Directory Purposes

**Project Root (`BTC_AutoDCA_bot/`):**
- Purpose: Holds all runtime code modules; no `src/` subpackage split.
- Contains: Top-level Python modules, docs, runtime artifact directories, and `.planning/`.
- Key files: `bot.py`, `auto_send.py`, `erc20.py`, `wallet.py`, `networks.py`, `requirements.txt`.

**Planning Directory (`.planning/`):**
- Purpose: Stores planning/mapping metadata for GSD workflow.
- Contains: Generated analysis docs in `.planning/codebase/`.
- Key files: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`.

**Runtime State Directory (`logs/`):**
- Purpose: Bot operational logs and lock execution markers.
- Contains: `logs/bot.log` (runtime), `logs/bot.lock`, `logs/last_seen_execution.txt`.
- Key files: `logs/bot.lock`, `logs/last_seen_execution.txt`.

**Wallet Storage Directory (`keystores/`):**
- Purpose: Persist encrypted keystore JSON files per user.
- Contains: Files generated by `generate_keystore_path` / `save_keystore` in `wallet.py`.
- Key files: Runtime-generated `keystores/user_<telegram_user_id>_wallet.json`.

## Key File Locations

**Entry Points:**
- `bot.py`: Process entry (`if __name__ == "__main__"`), `main`, bot startup, and task scheduling.
- `bot.py`: Telegram command registration via `@dp.message(...)`.

**Configuration:**
- `requirements.txt`: Runtime dependencies (`aiogram`, `aiosqlite`, `web3`, `eth-account`, `keyring`, etc.).
- `networks.py`: Mainnet/testnet chain config maps and network helper functions.
- `.env.example`: Required environment variable template for local setup.
- `.gitignore`: Runtime and secret exclusion policy (`.env`, `*.db`, `keystores/`, `wallet.json`, `venv/`).

**Core Logic:**
- `bot.py`: DCA plan lifecycle, DB schema/migrations, FixedFloat API client wrappers, scheduler, monitor, Telegram UX.
- `auto_send.py`: Safe auto-send pipeline with wallet/network locking and preflight checks.
- `erc20.py`: RPC fallback, gas parameter construction, approve/transfer execution, balance/allowance read operations.
- `wallet.py`: Keystore creation/loading/deletion and keyring password persistence.

**Testing:**
- Not detected: no `tests/` directory and no `test_*.py` / `*_test.py` files under project root.
- Manual guidance exists in `TESTING.md`.

## Naming Conventions

**Files:**
- Pattern: top-level snake_case module names.
- Examples: `auto_send.py`, `wallet.py`, `networks.py`.

**Directories:**
- Pattern: lowercase names for runtime/system folders.
- Examples: `logs/`, `keystores/`, `.planning/`.

## Where to Add New Code

**New Feature (bot behavior + user command):**
- Primary code: add handler and orchestration in `bot.py` near existing `@dp.message(...)` command sections.
- Persistence updates: extend `init_db` and related SQL paths in `bot.py`.
- Tests: create a new `tests/` directory (not present currently), then add feature tests there.

**New Blockchain or Token Operation:**
- Implementation: add low-level RPC/token functions in `erc20.py`.
- Orchestration call site: connect from `auto_send.py` if it belongs to send pipeline, then invoke from `bot.py`.

**New Wallet/Credential Behavior:**
- Implementation: `wallet.py` for keystore/keyring operations.
- Integration points: `cmd_setwallet`, `cmd_deletewallet`, startup password preload (`load_passwords_at_startup`) in `bot.py`.

**New Network/Chain Support:**
- Primary definitions: extend `NETWORKS_MAINNET` / `NETWORKS_TESTNET` in `networks.py`.
- Mapping updates: align FixedFloat symbol maps in `bot.py` (`NETWORK_CODES`, `FIXEDFLOAT_ASSET_MAP`).

**Utilities:**
- Shared helpers currently live in `bot.py`; place formatting/validation helpers close to existing helper section in `bot.py`.
- If helper growth increases complexity, extract to a new top-level module (snake_case) and import into `bot.py`.

## Special Directories

**`logs/`:**
- Purpose: Runtime operational state and lock/log files used by `bot.py`.
- Generated: Yes.
- Committed: Partially (tracked files include `logs/bot.lock` and `logs/last_seen_execution.txt`; log files like `logs/bot.log` are ignored).

**`keystores/`:**
- Purpose: Encrypted wallet material at runtime.
- Generated: Yes.
- Committed: No (`keystores/` is gitignored in `.gitignore`).

**`venv/`:**
- Purpose: Local Python environment.
- Generated: Yes.
- Committed: No (`venv/` is gitignored in `.gitignore`).

**`.planning/codebase/`:**
- Purpose: Machine-authored architecture/structure references for planning/execution commands.
- Generated: Yes.
- Committed: Yes (intended planning artifacts).
